# FAQ

---

**Q: Это легально?**

Статья носит технический характер. aiway решает проблему геоблокировок, наложенных иностранными сервисами (OpenAI, Google, Anthropic) на российских пользователей. Метод (SNI Proxy) технически непригоден для обхода ТСПУ — он не скрывает имя домена, SNI передаётся открытым текстом.

Перед использованием ознакомьтесь с актуальным законодательством РФ самостоятельно.

---

**Q: Мой интернет не замедлится?**

Только для AI-сервисов, и незначительно. Всё остальное идёт как обычно — напрямую. DNS-запросы к Blocky добавляют ~30-50 мс (если VPS в Европе), что заметно только при открытии первой страницы на сайте. Скорость загрузки контента от этого не зависит.

---

**Q: Нужно что-то устанавливать на телефон/ноутбук?**

Нет. Достаточно прописать IP вашего VPS как DNS-сервер. Можно сделать один раз на роутере — тогда все устройства в сети получат настройки автоматически.

---

**Q: ChatGPT показывает капчи / банит за подозрительную активность**

Это случается если с одного IP одновременно используют прокси несколько человек. Решение: не раздавать доступ к вашему VPS слишком широко. Для личного использования проблем обычно не возникает.

---

**Q: Работает ли это с мобильным интернетом (не Wi-Fi)?**

Да, если прописать DNS на уровне конкретного соединения (не роутера). На Android: настройки Wi-Fi → IP: статичный → DNS. На iOS аналогично. Или использовать DoT/DoH — тогда DNS зашифрован и работает поверх любого соединения.

---

**Q: Blocky упал / контейнер не запускается**

```bash
docker logs blocky --tail 50
```

Чаще всего причина — порт 53 занят чем-то ещё. Проверяем:

```bash
ss -tlunp | grep :53
```

Если там `systemd-resolved` — `fix_resolved()` должна была его отключить. Если там что-то другое — останавливаем вручную и перезапускаем `install.sh`.

---

**Q: Angie не стартует после установки**

```bash
angie -t           # покажет строку с ошибкой в конфиге
journalctl -u angie -n 30
```

Чаще всего причина — порт 443 или 80 занят другим веб-сервером (apache2, nginx). Останавливаем:

```bash
systemctl stop apache2 nginx
systemctl disable apache2 nginx
```

---

**Q: DNS работает, но сайт всё равно не открывается**

Проверяем, что DNS резолвится правильно:

```bash
dig openai.com @ВАШ_VPS_IP +short
# должен вернуть IP вашего VPS
```

Если IP правильный, но сайт не открывается — скорее всего, порт 443 закрыт в файрволе VPS-провайдера. Откройте в панели управления VPS.

---

**Q: GitHub не работает через aiway — нужно ли это?**

Необязательно. GitHub в России обычно доступен напрямую. Если добавляете `github.com` в список — следите, что это не замедлит git-операции. Copilot (`githubcopilot.com`) стоит добавить отдельно, если он нужен.

---

**Q: Могу ли я добавить свой домен (например, новый AI-стартап)?**

Да, это основной сценарий. Открываете `lib/domains.sh`, добавляете домен в оба массива (`AI_DOMAINS` и `AI_APEX_DOMAINS`), перезапускаете `sudo bash install.sh`. Конфиги перегенерируются автоматически.

---

**Q: Трафик расшифровывается на VPS?**

Нет. Angie использует `ssl_preread` — читает только поле SNI из первого пакета TLS (оно идёт открытым текстом по стандарту протокола), и пересылает весь поток как есть, не расшифровывая. TLS-соединение устанавливается между вашим браузером и настоящими серверами AI-сервиса.

---

**Q: Какой VPS выбрать?**

Минимальные требования: 512 МБ RAM, 1 vCPU, белый IPv4-адрес. Цена — от 3-5€ в месяц у европейских хостеров.

Важен **IP-репутация**: на дешёвых серверах у «ноунейм» хостеров IP может быть уже забанен OpenAI/Google из-за предыдущих жильцов. Если ChatGPT сразу блокирует — пробуйте другого провайдера или другой сервер.

Локация: Нидерланды, Финляндия, Германия — хороший баланс пинга и доступности IP.

---

**Q: Я всё настроил, но Vision Pro / Apple TV не подхватывает DNS**

Apple-устройства в отдельных случаях игнорируют DNS из DHCP. Решение: прописать DNS вручную в настройках Wi-Fi конкретного соединения на устройстве.

---

**Q: Можно ли использовать с роутером Keenetic?**

Да. В настройках DHCP Keenetic укажите IP вашего VPS как DNS 1. Все устройства в сети автоматически получат его. Дополнительно можно настроить DoT на уровне роутера (Keenetic поддерживает DNS-over-TLS).
