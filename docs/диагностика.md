# Диагностика

Что-то не работает? Разбираемся по шагам.

---

## Быстрая проверка

```bash
# 1. Blocky запущен?
docker ps | grep blocky

# 2. DNS возвращает правильный IP?
dig openai.com @ВАШ_VPS_IP +short
# должен вернуть: ВАШ_VPS_IP

# 3. Angie запущен?
systemctl status angie

# 4. Порт 443 слушается?
ss -tlnp | grep :443

# 5. Порт 53 отвечает?
dig openai.com @127.0.0.1 +short   # с самого VPS
```

---

## Blocky

### Контейнер не запустился

```bash
docker logs blocky --tail 50
```

Типичные ошибки:

| Ошибка | Причина | Решение |
|:--|:--|:--|
| `address already in use :53` | Что-то занимает порт 53 | `ss -tlunp \| grep :53`: найти и остановить процесс |
| `permission denied` | Нет прав на бинд порта <1024 | Запускать от root или дать capability `NET_BIND_SERVICE` |
| `config file not found` | Нет `/opt/blocky/config.yml` | Перезапустить `install.sh` |

```bash
# Кто занимает порт 53?
ss -tlunp | grep :53
fuser 53/udp 53/tcp

# Остановить systemd-resolved вручную (если install.sh не отработал):
sed -i '/^#\?DNSStubListener=/d' /etc/systemd/resolved.conf
echo "DNSStubListener=no" >> /etc/systemd/resolved.conf
systemctl restart systemd-resolved
```

### DNS возвращает неправильный IP

```bash
# Тест прямо с VPS:
dig openai.com @127.0.0.1 +short

# Посмотреть конфиг:
cat /opt/blocky/config.yml | grep -A 5 customDNS

# Перезапустить Blocky:
docker restart blocky
sleep 3
dig openai.com @127.0.0.1 +short
```

### Живые логи Blocky

```bash
docker logs -f blocky
```

## Angie

### Не запускается

```bash
# Проверить синтаксис конфига:
angie -t

# Посмотреть ошибки:
journalctl -u angie -n 50 --no-pager
```

Типичные ошибки:

| Ошибка | Причина | Решение |
|:--|:--|:--|
| `bind() to 0.0.0.0:443 failed` | Порт 443 занят | `ss -tlnp \| grep :443`: найти и остановить (apache2, nginx) |
| `unknown directive "acme"` | Старая версия Angie без ACME | Обновить Angie: `apt-get update && apt-get upgrade angie` |
| `cannot load certificate` | Сертификат не получен | ACME требует доступный порт 80 и правильный DNS для домена |

```bash
# Кто занимает 443?
ss -tlnp | grep :443
lsof -i :443

# Остановить конкурентов:
systemctl stop nginx apache2
systemctl disable nginx apache2

# Перезапустить Angie:
systemctl restart angie
```

### Живые логи Angie

```bash
# Доступы (HTTP):
tail -f /var/log/angie/access.log

# Stream (SNI proxy):
tail -f /var/log/angie/stream.log

# Ошибки:
journalctl -u angie -f
```

## Сервис недоступен (DNS правильный, Angie работает)

### Проверяем SNI проброс

```bash
# Тест прямо с VPS через openssl:
openssl s_client -connect ВАШ_VPS_IP:443 -servername openai.com 2>&1 | head -20
# В выводе должно быть: CN = openai.com (не ваш домен)
```

Если видите CN вашего домена: что-то не так с конфигом stream блока.

### Проверяем файрвол

```bash
# С другой машины:
nc -zv ВАШ_VPS_IP 443
nc -zv ВАШ_VPS_IP 53

# На самом VPS:
iptables -L INPUT -n | grep -E "443|53"
ufw status
```

## Добавили домен, но он не работает

```bash
# 1. Проверяем, что домен есть в конфиге Blocky:
grep новый_домен /opt/blocky/config.yml

# 2. Если нет: запускаем install.sh заново (конфиги перегенерируются)
sudo bash install.sh

# 3. Проверяем резолюцию:
dig новый_домен @127.0.0.1 +short   # должен вернуть IP вашего VPS
```

## Сброс и переустановка

Если что-то пошло сильно не так: чистая переустановка:

```bash
# Удалить всё:
sudo bash uninstall.sh

# Установить заново:
sudo bash install.sh
```

`uninstall.sh` восстановит systemd-resolved из бэкапа, уберёт контейнер и конфиги Angie.

```bash

echo "=== Blocky ===" && docker ps | grep blocky
echo "=== Angie ===" && systemctl is-active angie
```
