# Настройка устройств

После установки aiway на VPS — нужно указать IP вашего VPS как DNS-сервер на устройствах. Больше ничего устанавливать не нужно.

---

## Стратегия: роутер vs. каждое устройство

**Рекомендуется: настроить на роутере**

Один раз меняете DNS в настройках DHCP роутера — все устройства в сети получают его автоматически. Телефоны, ноутбуки, телевизоры, игровые консоли — всё работает без дополнительных настроек.

**Резервный вариант: каждое устройство отдельно**

Если нет доступа к роутеру или нужно, чтобы работало на мобильном интернете — настраиваете DNS непосредственно на устройстве.

---

## Роутер (рекомендуется)

### Keenetic

```
Интернет → Сети и Wi-Fi → Домашняя сеть → DHCP
    DNS 1: ВАШ_VPS_IP
    DNS 2: 8.8.8.8  (резервный, на случай падения VPS)
```

Дополнительно можно настроить DoT (если у вас есть домен):
```
Интернет → DNS → DNS-over-TLS
    Имя сервера: dns.example.com
    Порт: 853
```

### OpenWrt

```bash
uci set dhcp.@dnsmasq[0].server="ВАШ_VPS_IP"
uci commit dhcp
/etc/init.d/dnsmasq restart
```

### MikroTik

```
IP → DHCP Server → Networks → DNS Servers → ВАШ_VPS_IP
```

## Android

### Вариант 1: Частный DNS (DoT) — рекомендуется для Android 9+

```
Настройки → Сеть и Интернет → Частный DNS
    Имя хоста частного DNS-сервера: dns.example.com
```

Работает поверх любого подключения, в том числе мобильного интернета. Зашифровано.

### Вариант 2: DNS для конкретной Wi-Fi сети

```
Настройки → Wi-Fi → (долгое нажатие на сеть) → Изменить сеть
    IP-настройки: Статичный
    DNS 1: ВАШ_VPS_IP
    DNS 2: 8.8.8.8
```

## iOS / iPadOS

### Wi-Fi

```
Настройки → Wi-Fi → (нажать ⓘ рядом с сетью)
    → Настроить DNS → Вручную
    → Удалить все существующие серверы
    → Добавить сервер: ВАШ_VPS_IP
```

### Мобильный интернет (через профиль DoH)

Apple не поддерживает "Частный DNS" как Android. Для работы через мобильный интернет нужно установить конфигурационный профиль `.mobileconfig` с настройкой DoH.

Создать профиль можно командой на VPS (если настроен домен для DoH):

```bash
# На вашем Mac/Linux:
cat > aiway.mobileconfig << EOF
<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE plist PUBLIC "-//Apple//DTD PLIST 1.0//EN" "http://www.apple.com/DTDs/PropertyList-1.0.dtd">
<plist version="1.0">
<dict>
    <key>PayloadContent</key>
    <array>
        <dict>
            <key>DNSSettings</key>
            <dict>
                <key>DNSProtocol</key>
                <string>HTTPS</string>
                <key>ServerURL</key>
                <string>https://dns.example.com/dns-query</string>
            </dict>
            <key>PayloadType</key>
            <string>com.apple.dnsSettings.managed</string>
            <key>PayloadUUID</key>
            <string>$(uuidgen)</string>
            <key>PayloadIdentifier</key>
            <string>com.aiway.dns</string>
            <key>PayloadVersion</key>
            <integer>1</integer>
        </dict>
    </array>
    <key>PayloadType</key>
    <string>Configuration</string>
    <key>PayloadUUID</key>
    <string>$(uuidgen)</string>
    <key>PayloadIdentifier</key>
    <string>com.aiway.config</string>
    <key>PayloadVersion</key>
    <integer>1</integer>
    <key>PayloadDisplayName</key>
    <string>aiway DNS</string>
</dict>
</plist>
EOF
```

Переносите на iPhone через AirDrop или почту, устанавливаете через «Настройки → Профиль загружен».

## macOS

```
Системные настройки → Сеть → Wi-Fi → Подробнее → DNS
    Нажать "+" → ввести ВАШ_VPS_IP
    Переместить вверх списка (перетащить)
    Нажать OK → Применить
```

> **Важно:** не удаляйте остальные DNS-серверы полностью — оставьте `8.8.8.8` как резервный на случай недоступности VPS.

---

## Windows 10 / 11

```
Параметры → Сеть и Интернет → Wi-Fi → (ваша сеть) → Свойства
    Назначение DNS-сервера: Вручную
    IPv4: Включить
    Предпочитаемый DNS: ВАШ_VPS_IP
    Альтернативный DNS: 8.8.8.8
```

## Linux

### Временно (до перезагрузки)

```bash
echo "nameserver ВАШ_VPS_IP" | sudo tee /etc/resolv.conf
```

### Через systemd-resolved (постоянно)

```bash
# /etc/systemd/resolved.conf
[Resolve]
DNS=ВАШ_VPS_IP
FallbackDNS=8.8.8.8
```

```bash
sudo systemctl restart systemd-resolved
```

### Через NetworkManager

```bash
nmcli con mod "Имя сети" ipv4.dns "ВАШ_VPS_IP"
nmcli con up "Имя сети"
```

## Apple Vision Pro

Vision Pro иногда игнорирует DNS из DHCP роутера. Если AI-сервисы не работают — прописываем вручную:

```
Настройки → Wi-Fi → (ваша сеть) → (ⓘ)
    → Настроить DNS → Вручную
    → Добавить ВАШ_VPS_IP
```

## Проверка

После настройки DNS — убедитесь, что резолюция работает правильно:

**macOS / Linux:**
```bash
dig openai.com +short
# должен вернуть IP вашего VPS

nslookup openai.com
# Server: ВАШ_VPS_IP
# Address: ВАШ_VPS_IP

# Проверяем, что обычные сайты не сломались:
dig yandex.ru +short
# должен вернуть настоящий IP Яндекса, не ваш VPS
```

**Онлайн (с телефона):** открываем `chatgpt.com` — если загружается без ошибки "Access Denied", всё работает.
